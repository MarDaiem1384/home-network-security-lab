\# Wireshark Traffic Analysis – Initial Capture



\## Objective

To capture and analyse network traffic in order to establish a baseline of normal activity within a home network.



\## Environment

\- Device: Windows 11 PC

\- Network: Home Wi-Fi

\- Tool: Wireshark



\## Capture Summary

A short packet capture was conducted during normal browsing activity.



\## Observed Protocols

\- DNS – Domain name resolution requests

\- TCP – Transport protocol for web traffic

\- TLS/HTTPS – Encrypted communication

\- ARP – Local network communication



\## Key Observations

\- DNS requests observed when accessing websites

\- TLS traffic indicates encrypted connections

\- No suspicious or unexpected traffic identified

\- ARP activity consistent with local network communication



\## Assessment

Traffic appears consistent with normal user behaviour.



No indicators of compromise identified during this initial capture.



\## Limitations

\- Short capture duration

\- Single device scope

\- No deep packet inspection performed



\## Next Steps

\- Capture longer sessions

\- Identify external IP connections

\- Monitor for unusual patterns



\## Learning Outcome

Developed understanding of baseline network behaviour and common protocols.
